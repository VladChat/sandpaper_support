# Taxonomy

## Goal
Define the canonical support taxonomy for the project.

The taxonomy should help us keep:
- navigation consistent
- search mapping consistent
- page generation consistent
- AI routing consistent

---

## 1) Problem taxonomy

### Top-level problems
- scratches-too-deep
- not-sure-what-grit-to-use
- paper-clogs-too-fast
- sanding-takes-too-long
- finish-looks-uneven
- swirl-marks-remain
- paper-tears-early
- poor-results-between-coats
- wet-sanding-leaves-haze
- surface-still-feels-rough

## 3) Grit taxonomy

### Grit families
- coarse
- medium
- fine
- extra-fine
- ultra-fine

### Suggested family mapping
- coarse: 60, 80, 100, 120
- medium: 150, 180, 220, 240
- fine: 280, 320, 360, 400
- extra-fine: 500, 600, 800
- ultra-fine: 1000, 1200, 1500, 2000, 3000

## 6) Search language rules
Use user language first.

Examples:
- "scratches too deep"
- "paper clogging"
- "what grit for wood"
- "haze after wet sanding"
- "surface still rough"

Avoid making the taxonomy dependent on internal-only labels.

---

## 7) Canonical naming rules
- use lowercase slugs
- use hyphens in slugs
- keep slugs stable once published
- prefer one canonical node for each major intent
- map synonyms through aliases, not duplicate pages
